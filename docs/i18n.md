# Internationalization (i18n)

The framework provides a lightweight i18n system based on React context. Translation keys in JSON page configs are automatically resolved at render time.

## Setup

Wrap your app in `I18nProvider` with the current locale and a translations map:

```tsx
import { I18nProvider } from 'pageforge';

const TRANSLATIONS = {
  en: { 'page.networks.title': 'Networks', 'col.common.name': 'Name' },
  es: { 'page.networks.title': 'Redes', 'col.common.name': 'Nombre' },
};

function App() {
  const [locale, setLocale] = useState('en');
  return (
    <I18nProvider locale={locale} translations={TRANSLATIONS}>
      <AppContent />
    </I18nProvider>
  );
}
```

### Props

| Prop | Type | Description |
|------|------|-------------|
| `locale` | `string` | Current locale key (e.g. `'en'`, `'es'`, `'pt-BR'`) |
| `translations` | `Record<string, Translations>` | Map of locale → key-value translation pairs |
| `children` | `React.ReactNode` | App content |

`Translations` is typed as `Record<string, string>`.

## Translation Files

Create one file per locale, each exporting a flat `Record<string, string>`:

```typescript
// app/i18n/en.ts
export const en: Record<string, string> = {
  // Navigation
  'nav.dashboard': 'Dashboard',
  'nav.networks': 'Networks',

  // Page titles
  'page.networks.title': 'Networks',
  'page.networks.description': 'Manage and configure your networks.',

  // Column labels
  'col.networks.id': 'Network ID',
  'col.common.name': 'Name',
  'col.common.status': 'Status',

  // Actions
  'action.networks.create': 'Create network',
  'action.networks.delete': 'Delete network',

  // Framework defaults
  'list.loading': 'Loading...',
  'list.searchPlaceholder': 'Search...',
  'form.cancel': 'Cancel',
  'form.submit': 'Submit',
};
```

```typescript
// app/i18n/es.ts
export const es: Record<string, string> = {
  'nav.dashboard': 'Panel',
  'nav.networks': 'Redes',
  'page.networks.title': 'Redes',
  'page.networks.description': 'Administre y configure sus redes.',
  'col.networks.id': 'ID de red',
  'col.common.name': 'Nombre',
  'col.common.status': 'Estado',
  'action.networks.create': 'Crear red',
  'action.networks.delete': 'Eliminar red',
  'list.loading': 'Cargando...',
  'list.searchPlaceholder': 'Buscar...',
  'form.cancel': 'Cancelar',
  'form.submit': 'Enviar',
};
```

Then create an index that aggregates all locales:

```typescript
// app/i18n/index.ts
import { en } from './en';
import { es } from './es';

export const TRANSLATIONS: Record<string, Record<string, string>> = { en, es };

export const LANGUAGE_OPTIONS = [
  { id: 'en', text: 'English' },
  { id: 'es', text: 'Español' },
];
```

## Using Translations

The `useI18n` hook provides the `t` function and current `locale`:

```tsx
import { useI18n } from 'pageforge';

function MyComponent() {
  const { t, locale } = useI18n();

  return (
    <div>
      <h1>{t('page.networks.title', 'Networks')}</h1>
      <p>Current locale: {locale}</p>
    </div>
  );
}
```

### `t(key, fallback?)`

| Parameter | Type | Description |
|-----------|------|-------------|
| `key` | `string` | Translation key to look up |
| `fallback` | `string` (optional) | Returned if no translation exists. Defaults to the key itself. |

Lookup order:
1. `translations[locale][key]` — if found, return it
2. `fallback` — if provided, return it
3. `key` — return the raw key string

## JSON Config i18n

Framework components (`ListPage`, `FormPage`) automatically pass config string values through `t()`. Use translation keys directly in your JSON configs:

```json
{
  "title": "page.networks.title",
  "columns": [
    { "key": "id", "label": "col.networks.id" }
  ],
  "actions": [
    { "label": "action.networks.create", "action": "create", "variant": "primary" }
  ]
}
```

At render time, `ListPage` calls `t('col.networks.id', 'col.networks.id')` for each label. If a translation exists, it's displayed. Otherwise, the key itself appears — making it easy to spot missing translations.

Select option labels in form configs are also translated:

```json
{
  "key": "country",
  "type": "select",
  "label": "field.profile.country",
  "options": [
    { "label": "option.profile.country.US", "value": "US" },
    { "label": "option.profile.country.BR", "value": "BR" }
  ]
}
```

## Adding a Language

1. Create a new locale file:

```typescript
// app/i18n/pt-BR.ts
export const ptBR: Record<string, string> = {
  'nav.dashboard': 'Painel',
  'nav.networks': 'Redes',
  'page.networks.title': 'Redes',
  'col.networks.id': 'ID da rede',
  'col.common.name': 'Nome',
  // ... all keys
};
```

2. Register it in the index:

```typescript
// app/i18n/index.ts
import { en } from './en';
import { es } from './es';
import { ptBR } from './pt-BR';

export const TRANSLATIONS: Record<string, Record<string, string>> = {
  en, es, 'pt-BR': ptBR,
};

export const LANGUAGE_OPTIONS = [
  { id: 'en', text: 'English' },
  { id: 'es', text: 'Español' },
  { id: 'pt-BR', text: 'Português (BR)' },
];
```

3. The language is now available. If you have a language switcher (see below), it appears automatically.

## Key Naming Convention

Use dot-separated prefixes to organize translation keys:

| Prefix | Usage | Example |
|--------|-------|---------|
| `page.*` | Page titles and descriptions | `page.networks.title` |
| `col.*` | Table column headers | `col.networks.id` |
| `action.*` | Button labels | `action.networks.create` |
| `field.*` | Form field labels | `field.profile.name` |
| `option.*` | Select option labels | `option.profile.country.US` |
| `nav.*` | Navigation items | `nav.networks` |
| `list.*` | Framework list defaults | `list.loading` |
| `form.*` | Framework form defaults | `form.cancel` |
| `detail.*` | Detail page labels | `detail.title` |
| `dashboard.*` | Dashboard widgets | `dashboard.networks` |

## Complete Example

### Language Switcher with TopNavigation

```tsx
// app/routes/_app.tsx
import { useState } from 'react';
import { Outlet, useLocation, useNavigate } from '@remix-run/react';
import TopNavigation from '@cloudscape-design/components/top-navigation';
import AppLayout from '@cloudscape-design/components/app-layout';
import SideNavigation from '@cloudscape-design/components/side-navigation';
import '@cloudscape-design/global-styles/index.css';
import { I18nProvider, useI18n } from 'pageforge';
import { TRANSLATIONS, LANGUAGE_OPTIONS } from '~/i18n';

function AppContent({ locale, setLocale }: { locale: string; setLocale: (l: string) => void }) {
  const { t } = useI18n();
  const location = useLocation();
  const navigate = useNavigate();

  return (
    <>
      <TopNavigation
        identity={{ href: '/', title: 'My Console' }}
        i18nStrings={{ overflowMenuTriggerText: 'More' }}
        utilities={[
          {
            type: 'menu-dropdown',
            text: LANGUAGE_OPTIONS.find(l => l.id === locale)?.text ?? 'English',
            items: LANGUAGE_OPTIONS.map(l => ({ id: l.id, text: l.text })),
            onItemClick: ({ detail }) => setLocale(detail.id),
          },
        ]}
      />
      <AppLayout
        toolsHide
        navigation={
          <SideNavigation
            activeHref={location.pathname}
            items={[
              { type: 'link', text: t('nav.dashboard'), href: '/' },
              {
                type: 'section',
                text: t('nav.manageNetworks'),
                items: [
                  { type: 'link', text: t('nav.networks'), href: '/networks' },
                ],
              },
            ]}
            onFollow={(e) => { e.preventDefault(); navigate(e.detail.href); }}
          />
        }
        content={<Outlet />}
      />
    </>
  );
}

export default function AppLayoutRoute() {
  const [locale, setLocale] = useState('en');
  return (
    <I18nProvider locale={locale} translations={TRANSLATIONS}>
      <AppContent locale={locale} setLocale={setLocale} />
    </I18nProvider>
  );
}
```

The language dropdown in `TopNavigation` lets users switch locales. All framework components and translated navigation items update immediately.
